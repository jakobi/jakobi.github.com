C02 - Another Way of Helping Android Cope with Moore's Law

Last change: 20120216

Still polishing notes - version 1 to be online before 20120220

I  cannot help but assume that Android devices were never  expected
to  grow  to  Deskop  size, considering  user  interface,  application
presentation  and  some sneak peeks into the framework  source.  Maybe
Google  wanted to be polite and offer the user to install a dozen apps
of his own.

But  a  modern  Android runs on  different  hardware:  Desktop-size
displays  (1280x800)  coupled  with Desktop-size RAM (1GB)  and  Flash
sizes  already  suitable  to  a full-sized  Linux  or  Windows  "root"
installation. Available space and possibilities tend to be used, so we
have  standard  Android setups using 180 apps as part of the rom  blob
with maybe 200 to 300 additional installs from the user.

This  article describes an approach to use the default app launcher
of  the  note to organize your apps and to reuse that data outside  of
the  launcher,  e.g.  in  conjunction with  titanium  backup  and  apk
management after e.g. flashing a new rom.
