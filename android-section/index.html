Coping with Size

I cannot help but assume that Android devices were never expected to grow to Deskop size, considering user interface, application presentation and some sneak peeks into the framework source. Maybe be polite and offer the user to install a dozen apps of his own. But a modern Android runs on different hardware: Desktop-size displays (1280x800) coupled with Desktop-size RAM (1GB) and Flash sizes already suitable to a full-sized Linux or Windows "root" installation. This results in setups using about 200 apps as part of the rom and maybe 200 to 300 additional installs from the user. This article describes an approach to use the default app launcher of the note to organize your apps and to reuse that data outside of the launcher, e.g. in conjunction with titanium backup and apk management after e.g. flashing a new rom.

A look at the privacy and security implications of smartphones, using the Galaxy Note as an example
