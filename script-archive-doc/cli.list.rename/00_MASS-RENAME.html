Peter's Homepage - Mass-Renaming Files

This toolbox compartment offers two tools for mass renaming files, both of which also offering UNDO-loggings and more.

One allows for way too much configurability and perl evals with an optional interactive mode that acts on each file independently (myrename), while the other shows all files at once in your favorite editor (emv). If the task is repetitive, use myrename; if you need to do it all the time, pre-define a small perl scrap for myrename and just say myrename <NAME> FILES. If you want to see all the files at once and want to think about their names while editing and changing whole groups of files, use emv instead (when using vim, do read up on g//p, sort, regex and visual blocks; note that emv is rather useless when combined with a weak editor).

See also GREP.xchange, which is somewhat similar:

  • file list -> grep output
  • mass-rename -> mass edit
  • ls -1d FILES | emv - -> grep FILES | pipe.vim | Grep.xchange

Commands

  • emv (perl) - edited mv: two pass renaming, link creation, file list splitting, link target editing (use e.g. GNU find -type l -xtype l | emv -); quite capable when used with the vim editor's visual block changes. Can also be used non-interactively by providing a perl expression for renaming files.
  • myrename (perl): mass renaming using a perl expression like e.g. s/old/new/, also allowing for predefined actions and extensive user customization of actions, e.g. things like myrename lc to lowercase all files that match the shell pattern *. With myrename '$l=~s/foo/bar' FILES all link matching link targets are modified. Way too many evaluation hooks for customization. Features a basic interactive mode. A configuration example is also provided.

Other commands:

  • emvs, a helper to sort filelists but keeping directories and contents always together, temporarily remapping / to \0.
  • myrename.mangle: parse filelists using shell quoting rules and encode/decode names for e.g. sorting; from whitespace elimination to url-style-encoding.
  • myrename.quote: allow the use of the quoting routines from emv/myrename for quoting filenames safely for use in shell commands.
