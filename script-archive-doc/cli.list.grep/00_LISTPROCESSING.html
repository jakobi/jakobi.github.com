Peter's Homepage - List Processing

This toolbox compartment contains the heavy lifting tools for generic pipeline 'line'-based text processing.

Commands

  • cat0 converts from and to \0-terminated lines. Additional wrappers implementing support for this format: nl0. The GNU tools, grep, sort, xargs already support this format, thus I've not yet needed/created a sort0; and xargs0 is still just a glorified exec to reduce typos. As for grep, I'd prefer Grep.pm -0 below.
  • cat0par converts paragraphs into \0 lines and also allows for arbitrary regex-based splitting. An overgrep perl --0ne/perl -0777ne.
  • Grep.pm is a fork of tcgrep, which nowadays has learned about context, matching from start or end of file (line count or byte count), size limits, and highlighting. It extends matching to boolean expressions structuring regular expressions or even arbitrary perl scraps. It can perform basic stemming and synonym-expansion in regular expressions (using expansyn). It also handles \0-lines, paragraphs and file slurping, as well as directory recursion and compressed files (both as in tcgrep). Dependencies: expansyn. It can masquerade as a perl module.
  • How about a kind of emv not for mass-renaming filenames, but for mass-editing file-contents?
    Grep.xchange bridges the gap between the grep and patch commands: it takes grep output and a perl expression and uses both to change an arbitrary number of files by applying the expression at each of the grep matches, while generating a patch file as its UNDO / REDO log. It's a bit paranoid, so combined with edited grep output (or Grep.pm -C -LINES marked grep context output), the 3-Step Grep.pm|vim.pipe|Grep.xchange is a rather safe way to mass-edit files. Dependencies: For flock-locking edited files, you also need to copy Compact_pm::Flock.pm; otherwise you can disable it by commenting the use statement.
  • expansyn is the command implementing basic English stemming and synonym expansion for Grep.pm / tagls using a configuration file expansyn.cfg containing the synonymlists in perl syntax. I wrote it for automatic expansion of some computer science terms I always seem to use in parallel.

See also:

  • f2a in 00_lib.func: a shell function reading a file into an array: find . -print | f2a -a ARGS; less "$ARGS[@]"
  • pipe.vim (aka vimpipe) , vimless to run vim in the middle of a pipe or as a less replacement. Both are in the vim section.
  • Alternatives for automatic mass file editing are also the usual suspects: stream editors from sed to perl -i.bak and the perl ped script, selectlines, xchange.pl (both pretty ancient and available elsewhere in this archive) upto full-featured administration tools like cfengine with their support to modify or generate configuration files. At the beginning of Grep.xchange you also find two interative perl one-liner examples that interactively ask before each substitution.
